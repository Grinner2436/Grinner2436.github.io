# ES11：缓存和冲刷


### Flush
保证将数据从 [transaction log]刷到Lucene
```
POST twitter/_flush?wait_if_ongoing
wait_if_ongoing：前一个刷新没执行，是否等待
```

### Refresh
使上次refresh之后的所有操作变得可搜索
```
POST  /twitter/_refresh
```
#### 1. 支持?refresh参数的请求接口
文档Document的Index, Update, Delete, Bulk支持刷新。

#### 2. 选项
  * Empty string or `true` ：立即刷新索引，使最新修改可搜索。
  
  * wait_for ：直到出现一次显式的refresh，再刷新索引。
   
  * `false` (the default) : 不采取任何操作，直到请求接口返回。
#### 3. 如何选择 
  立即刷新会造成索引碎片。导致索引、搜索、合并时的性能损耗。
  尽量不开启刷新。
